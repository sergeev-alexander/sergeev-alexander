# Testing theory

<details>
    <summary>
        <b>Типы/виды тестирования</b>
    </summary>

### ТИПЫ ТЕСТИРОВАНИЯ (по уровню доступа к коду/системе)

- `Black Box` (Черный ящик): Тестирование без знания внутреннего устройства системы, основанное на требованиях и спецификациях.
- `White Box` (Белый ящик): Тестирование с полным знанием исходного кода (`source code`) и внутренней структуры системы.
- `Grey Box` (Серый ящик): Тестирование с частичным знанием внутреннего устройства, часто на уровне взаимодействия модулей или `API`.

### ТИПЫ ТЕСТИРОВАНИЯ (по исполнению кода)

- `Static Testing` (Статическое тестирование): Анализ артефактов (код, требования, дизайн) без запуска программы.
- `Dynamic Testing` (Динамическое тестирование): Проверка поведения программы во время выполнения ее кода.

### ТИПЫ ТЕСТИРОВАНИЯ (по степени автоматизации)

- `Manual Testing` (Ручное тестирование): Выполнение тестовых сценариев (`test scenarios`) человеком без использования скриптов.
- `Automated Testing` (Автоматизированное тестирование): Выполнение тестов с помощью специальных программ и скриптов.

### ВИДЫ ТЕСТИРОВАНИЯ (Функциональные / `Functional`)

- `Functional Testing` (Функциональное тестирование): Проверка соответствия функциональности системы заявленным требованиям.
- `Security Testing` (Тестирование безопасности): Проверка способности системы противостоять злонамеренным попыткам получить доступ или вызвать сбой.
- `Interoperability` / `Integration Testing` (Тестирование взаимодействия / Интеграционное): Проверка связи между компонентами или с внешними системами.

### ВИДЫ ТЕСТИРОВАНИЯ (Нефункциональные / `Non-Functional`)

- `Performance Testing` (Тестирование производительности): Оценка скорости, времени отклика (`response time`), стабильности и потребления ресурсов под нагрузкой.
- `Load Testing` (Нагрузочное тестирование): Оценка поведения системы под ожидаемой или пиковой нагрузкой (`peak load`).
- `Stress Testing` (Стресс-тестирование): Оценка поведения системы под нагрузкой, превышающей пределы нормальной работы.
- `Volume Testing` (Объемное тестирование): Проверка работы системы с большими объемами данных (`data volumes`).
- `Installation`/`Deployment Testing` (Тестирование установки/развертывания): Проверка корректности процессов установки, настройки, обновления и удаления ПО.
- `Usability Testing` (Тестирование удобства использования): Оценка того, насколько продукт понятен, удобен и привлекателен для пользователя.
- `Failover and Recovery Testing` (Тестирование на отказ и восстановление): Проверка способности системы восстанавливаться после аппаратных или программных сбоев.
- `Configuration` / `Compatibility Testing` (Конфигурационное / Тестирование совместимости): Проверка работы системы на различном окружении (ОС, браузеры, устройства и т.д.).

### ВИДЫ ТЕСТИРОВАНИЯ (Связанные с изменениями / `Change-Related`)

- `Smoke Testing` (Дымовое тестирование): Поверхностная проверка основных функций (`core functions`) для подтверждения готовности сборки (`build`) к дальнейшему тестированию.
- `Regression Testing` (Регрессионное тестирование): Повторное тестирование для подтверждения того, что ранее работающая функциональность не сломана после изменений.
- `Sanity Testing` (Санитарное тестирование): Узконаправленная проверка одной конкретной функции или исправления (`bug fix`) после изменений.
- `Build Verification Test` / `BVT` (Тестирование сборки): Аналог дымового тестирования для принятия решения о приемке новой версии в работу.

### УРОВНИ ТЕСТИРОВАНИЯ (`Testing Levels`)

- `Unit Testing` (Модульное / Компонентное тестирование): Тестирование отдельных изолированных модулей или компонентов (обычно методов или классов) программы.
- `Integration Testing` (Интеграционное тестирование): Тестирование взаимодействия между несколькими модулями, компонентами или системами.
- `System Testing` (Системное тестирование): Тестирование полностью интегрированной системы на соответствие всем требованиям.
- `Acceptance Testing` (Приемочное тестирование): Финальное тестирование, проводимое с целью принятия решения о готовности системы к выпуску.

### ДОПОЛНИТЕЛЬНО ДЛЯ `AQA` (Актуальные виды/подходы)

- `API Testing` (Тестирование `API`): Тестирование программных интерфейсов приложения (слой сервисов) без графического интерфейса (`GUI`).
- `Database` / `DB Testing` (Тестирование баз данных): Проверка целостности данных (`data integrity`), корректности запросов (`queries`), производительности БД.
- `Cross-Browser` / `Cross-Platform Testing` (Кросс-браузерное / Кроссплатформенное тестирование): Частный случай конфигурационного тестирования для веб и мобильных приложений.
- `Containerization Testing` (e.g., `Docker`) (Тестирование в контейнерах): Проверка работы приложения в изолированных контейнерных средах.
- `Test Automation` / `Writing Autotests` (Автоматизация тестирования / Написание автотестов): Разработка автоматизированных тестовых сценариев с использованием фреймворков (`JUnit`, `TestNG`, `Selenium`, `REST Assured` и т.д.).
