# 1. Введение

> XPath (XML Path Language) — язык запросов для навигации и выборки узлов в XML и HTML-документах. 
> В автотестировании он выступает мощным инструментом для точного поиска элементов, позволяя двигаться по DOM-дереву, фильтровать по тексту, атрибутам и отношениям между узлами.
>
> В отличие от CSS-селекторов, XPath поддерживает обратную навигацию (поиск родителей), работу с видимым текстом и сложные логические условия, что делает его незаменимым при тестировании динамических интерфейсов.

## Зачем XPath нужен в Selenium

- **Универсальность поиска** — находит элементы по атрибутам, точному/частичному тексту, порядковому номеру или положению в дереве.
- **Гибкая навигация** — оси `parent::`, `ancestor::`, `following-sibling::` позволяют строить локаторы от известных соседей или родителей.
- **Нативная поддержка** — Selenium предоставляет `By.xpath()` из коробки, без дополнительных библиотек или парсеров.
- **Стабильность в Selenoid** — Selenoid передаёт команды XPath напрямую в драйвер контейнеризированного браузера. 

  Поведение полностью идентично локальному запуску, так как вычисление происходит на стороне браузера, а не на стороне хаба.

## Встроенная поддержка в Selenium

- Selenium оборачивает XPath-выражение в объект `By.xpath(String expression)`.
- Возвращает интерфейс `WebElement`, поддерживающий стандартные действия: `.click()`, `.sendKeys()`, `.getAttribute()`, `.getText()`.
- Ошибки поиска выбрасываются как `NoSuchElementException`, если элемент не найден в текущем состоянии DOM.

```java
public class XPathIntroExample {
    
    public static void main(String[] args) {
        // Инициализация драйвера
        WebDriver driver = new ChromeDriver();
        
        try {
            driver.get("file:///path/to/form.html");
            
            // Поиск кнопки по точному тексту внутри тега
            WebElement loginButton = driver.findElement(
                By.xpath("//button[text()='Войти']")
            );
            
            // Поиск поля ввода по уникальному id
            WebElement emailField = driver.findElement(
                By.xpath("//input[@id='email']")
            );
            
            // Взаимодействие с найденными элементами
            emailField.sendKeys("test@example.com");
            loginButton.click();
            
        } finally {
            // Гарантированное закрытие сессии и освобождение ресурсов
            driver.quit();
        }
    }
}
```

## Базовая инициализация WebDriver для работы с XPath

- В локальных тестах используется конкретный драйвер (`ChromeDriver`, `FirefoxDriver`).
- В Selenoid применяется `RemoteWebDriver` с указанием URL хаба и `DesiredCapabilities`.
- XPath-локаторы передаются в методы поиска без модификаций, но требуют синхронизации с состоянием страницы через явные ожидания.

```java
public class SelenoidXPathInit {

    public static void main(String[] args) throws Exception {
        // Настройка браузера для запуска в Selenoid
        DesiredCapabilities caps = new DesiredCapabilities();
        caps.setBrowserName("chrome");
        caps.setCapability("version", "120.0");
        
        // Подключение к удалённому хабу
        WebDriver driver = new RemoteWebDriver(
            new URL("http://localhost:4444/wd/hub"),
            caps
        );
        
        try {
            driver.get("https://example.com/login");
            
            // Явное ожидание появления элемента перед поиском по XPath
            WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

            WebElement emailInput = wait.until(
                ExpectedConditions.visibilityOfElementLocated(
                    By.xpath("//input[@data-testid='password-field']")
                )
            );
            
            emailInput.sendKeys("securePassword123");
            
        } finally {
            driver.quit();
        }
    }
}
```

## Best Practices

- **Используйте относительные пути** (`//...`) вместо абсолютных (`/html/body/div/...`). Абсолютные пути ломаются при добавлении/удалении любого wrapper-элемента.
- **Опирайтесь на стабильные атрибуты**: `id`, `name`, `data-*` предпочтительнее `class`, которые часто меняются при обновлении UI-библиотек.
- **Комбинируйте XPath с ожиданиями**: мгновенный поиск `findElement()` падает, если DOM ещё не готов. Всегда используйте `WebDriverWait` в реальных тестах.
- **Валидируйте выражения до кода**: используйте `$x("//ваш_xpath")` в консоли DevTools. Это экономит время на запуск автотестов при подборе локатора.
- **Минимизируйте длину выражения**: чем короче XPath, тем он быстрее выполняется и устойчивее к рефакторингу вёрстки.

# 2. Основы синтаксиса и структура XPath-выражений

> XPath-выражения состоят из осей, узлов, предикатов и функций. 
> 
> Понимание базового синтаксиса позволяет создавать локаторы, которые не ломаются при изменении обёрток в вёрстке и остаются читаемыми для поддержки.

## Абсолютный и относительный пути

- **Абсолютный путь** начинается с корневого узла `/` и перечисляет всю иерархию до искомого элемента.
    - Пример: `/html/body/div/main/form/input`
    - **Когда уместен**: только для статичных XML-конфигов или парсинга документов с жёстко фиксированной структурой.
    - **Почему редко используется в автотестах**: добавление `<div>`, `<header>` или `<section>` ломает весь путь. Крайне высокая хрупкость.
- **Относительный путь** начинается с `//` и рекурсивно ищет элементы по всему документу, игнорируя глубину вложенности.
    - Пример: `//input[@type='text']`
    - **Основа устойчивых локаторов**: не зависит от позиции в DOM, реагирует только на атрибуты и теги самого элемента или его ближайших соседей.

### Пример HTML для демонстрации
```html
<!DOCTYPE html>
<html>
<body>
    <!-- //div[@class='wrapper'] — поиск обёртки по классу -->
    <div class="wrapper">
        <header>
            <!-- //h1[text()='Заголовок страницы'] — поиск по точному тексту -->
            <h1>Заголовок страницы</h1>
        </header>
        <main>
            <!-- //form[@id='loginForm'] — поиск формы по уникальному id -->
            <form id="loginForm">
                <!-- //input[@id='username'] — поиск по уникальному id (рекомендуется) -->
                <input type="text" id="username" name="user" class="form-input">
                <!-- //input[@name='pass'] — поиск по атрибуту name (стабильная альтернатива) -->
                <input type="password" id="pass" name="pass" class="form-input active">
                <!-- //button[text()='Войти'] — поиск кнопки по видимому тексту -->
                <button type="submit">Войти</button>
            </form>
        </main>
    </div>
</body>
</html>
```

## Синтаксис работы с атрибутами

- Обращение к атрибуту происходит через символ `@` сразу после имени тега или внутри предиката.
- Поддерживаются стандартные HTML-атрибуты и кастомные `data-*`.
- **Базовый формат**: `//тег[@атрибут='значение']`

### Примеры поиска по атрибутам

- `//input[@id='username']` — поиск по уникальному атрибуту `id` (самый быстрый и надёжный способ)
- `//input[@name='user']` — поиск по атрибуту `name` (стабилен при смене CSS-классов)
- `//button[@type='submit']` — поиск кнопки по типу действия
- `//input[@data-testid='email-field']` — поиск по кастомному атрибуту для тестов (индустриальный стандарт)
- `//div[@class='form-input']` — поиск по полному совпадению класса (⚠️ опасен, если в элементе несколько классов)

```html
<!-- Демонстрация работы с атрибутами -->
<input id="email" name="user_email" data-testid="email-field" class="form-control required">
<!-- //input[@id='email'] ✓ находит элемент -->
<!-- //input[@name='user_email'] ✓ находит элемент -->
<!-- //input[@data-testid='email-field'] ✓ находит элемент -->
<!-- //input[@class='form-control'] ✗ ⚠️ НЕ найдёт, т.к. в классе два значения через пробел -->
<!-- //input[@class='form-control required'] ✓ найдёт только при полном совпадении -->
```

## Предикаты и условия в квадратных скобках

- Предикаты `[...]` фильтруют выбранные узлы по индексу, атрибуту, тексту или функции.
- Индексы в XPath начинаются с **1**, а не с 0.

### Базовые предикаты

- `[1]` — первый элемент в выборке (глобально по документу при использовании `(//...)[1]`)
- `[@атрибут='значение']` — фильтр по точному совпадению атрибута
- `[last()]` — последний элемент в выборке
- `[position() > 1]` — все элементы, больше первого

### Комбинирование условий: `and`, `or`

- Внутри предиката можно логически объединять условия.
- `and`: все условия должны быть истинными одновременно.
- `or`: достаточно выполнения хотя бы одного условия.
- Операторы пишутся строчными буквами внутри квадратных скобок.

### Примеры комбинированных условий

- `//input[@type='text' and @name='user']` — `<input>` с типом `text` И именем `user`
- `//input[@id='username' or @name='user']` — `<input>` с id `username` ИЛИ name `user` (fallback-поиск)
- `//button[@type='submit' and not(@disabled)]` — активная кнопка отправки формы
- `//a[@href and @title]` — ссылка, у которой заполнены оба атрибута

```html
<!-- Пример для комбинированных условий -->
<form>
    <!-- //input[@type='text' and @name='search'] находит этот элемент -->
    <input type="text" name="search" placeholder="Поиск...">
    
    <!-- //input[@type='text' or @type='email'] находит оба <input> ниже -->
    <input type="text" name="login">
    <input type="email" name="contact">
    
    <!-- //button[not(@disabled)] находит только активную кнопку -->
    <button type="submit">Отправить</button>
    <button type="reset" disabled>Сбросить</button>
</form>
```

### Важное про индексы

- Запись `//input[2]` означает: «второй `input` внутри каждого родителя», а не «второй input на странице».
- Для глобального порядкового номера используйте скобки: `(//input)[2]`.

```html
<div class="section">
    <input name="first">  <!-- //div[@class='section']/input[1] -->
    <input name="second"> <!-- //div[@class='section']/input[2] -->
</div>
<div class="another">
    <input name="third">  <!-- //div[@class='another']/input[1] -->
</div>
```

- `//input)[2]` вернёт элемент с `name='second'` (глобально второй в документе)
- `//input[2]` вернёт элементы с `name='second'` и `name='third'` (второй в каждом родителе)


## Best Practices

- **Всегда используйте относительные пути** `//...`. Абсолютные пути ломаются при первом же рефакторинге обёрток в вёрстке.
- **Избегайте полного совпадения `@class`** если в HTML может быть несколько классов. Используйте `contains(@class, 'value')` для частичного поиска.
- **Ставьте индекс в скобках** `(//expr)[N]`, а не `//expr[N]`. Запись `//input[2]` означает «второй input внутри КАЖДОГО родителя», а не «второй input на странице».
- **Используйте `data-testid` или `data-qa`** для автотестов. Это изолирует логику поиска от CSS-стилей, фреймворковых обёрток и JS-рендеринга.
- **Помните о чувствительности к регистру**: `@Class` и `@class` — разные атрибуты. Значения атрибутов тоже регистрозависимы.
- **Проверяйте локаторы в DevTools** через `$x("//ваш_xpath")` перед переносом в код. Это экономит время на компиляцию и запуск Selenium.
- **Комбинируйте атрибуты для уникальности**: если `@class` нестабилен, добавьте `@type` или `@name`: `//input[@class='btn' and @type='submit']`.
